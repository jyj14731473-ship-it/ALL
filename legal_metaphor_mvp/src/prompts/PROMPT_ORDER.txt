Current Prompt Order

현재 유지되는 공식 실행 경로는 src/main.py의 full pipeline 하나다.
아래 파일만 현재 prompts 폴더에 남아 있으며, 삭제된 legacy candidate/staged 프롬프트는 더 이상 호출되지 않는다.

1) lemma_group_sanity.md
   - 호출 코드: src/preprocessing/lemma_group_sanity.py
   - 조건: --lemma-sanity-gpt가 켜져 있고 OPENAI_API_KEY가 있을 때만 LLM 사용
   - 입력: pos_nodes.json의 lemma_groups batch
   - 출력: pos_nodes_corrected.json
   - 역할: 형태소/POS/lemma group 결과를 은유 판정 전 안정적으로 정리한다.

2) contextual_meaning.md
   - 호출 코드: src/preprocessing/contextual_meaning.py
   - 조건: OPENAI_API_KEY가 있을 때 full pipeline에서 사용
   - 입력: pos_nodes_corrected.json의 lemma_groups와 source text
   - 출력: pos_nodes_contextualized.json
   - 역할: 각 lemma가 해당 문맥에서 어떤 의미로 쓰였는지 정한다.
   - DFMA/WIDLII 기준을 포함한다.

3) system_role.md
   - 호출 코드: src/nodes/mipvu_judge.py
   - 직접 단독 호출되지 않고 lemma_mipvu_judge.md의 system prompt 일부로 결합된다.
   - 역할: 법률 은유/MIPVU 주석자의 기본 역할을 지정한다.

4) korean_legal_mipvu_guideline.md
   - 호출 코드: src/nodes/mipvu_judge.py
   - 직접 단독 호출되지 않고 lemma_mipvu_judge.md의 system prompt 일부로 결합된다.
   - 역할: 한국어 법률 텍스트에 맞춘 MIPVU 판정 원칙을 제공한다.

5) lemma_mipvu_judge.md
   - 호출 코드: src/nodes/mipvu_judge.py
   - 입력:
     - data/output/lemma_dictionary_lookup.json
     - data/output/pos_nodes_contextualized.json
   - batch size: 25 lemma
   - 출력 위치:
     - data/output/graph_annotation.json 안의 mipvu_annotations
   - 역할:
     - 사전 뜻 후보 중 original/basic meaning을 선택한다.
     - contextual meaning과 original meaning을 비교해 MIPVU label을 판단한다.
     - MRW 계열이면 source_domain, target_domain, conceptual_metaphor, Lakoff-Johnson type을 함께 산출한다.

Non-Prompt Steps

아래 단계는 현재 LLM 프롬프트를 쓰지 않는다.

1) dictionary lookup
   - 호출 코드: src/preprocessing/dictionary_lookup.py
   - API: 표준국어대사전
   - 출력: data/output/lemma_dictionary_lookup.json

2) RDF mapping
   - 호출 코드: src/nodes/rdf_mapping.py
   - 입력: mipvu_annotations
   - 출력: state["rdf_mappings"]

3) validation
   - 호출 코드: src/nodes/validation_check.py
   - 입력: mipvu_annotations, rdf_mappings
   - 출력: validation_results, human_review_items

4) Turtle conversion
   - 호출 코드: src/nodes/rdf_convert.py, src/rdf/convert.py
   - 출력:
     - state["rdf_output"]
     - data/output/metaphors.ttl

5) RDFLib KG build
   - 호출 코드: src/graph_build.py
   - 출력: data/output/metaphor_kg.ttl

Full Order

data/input.txt
-> pos_nodes.json
-> lemma_group_sanity.md
-> pos_nodes_corrected.json
-> contextual_meaning.md
-> pos_nodes_contextualized.json
-> dictionary lookup
-> system_role.md + korean_legal_mipvu_guideline.md + lemma_mipvu_judge.md
-> graph_annotation.json / mipvu_annotations
-> rdf_mapping.py
-> validation_check.py
-> rdf_convert.py / rdf.convert
-> metaphors.ttl
-> graph_build.py
-> metaphor_kg.ttl
